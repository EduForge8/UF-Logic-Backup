##门店的员工保护协议

在 GitHub 的逻辑里，我们不靠“忍受”魔丸，我们靠 “逻辑对冲” 保护员工：

1. 逻辑对冲：传感器是不会撒谎的（Data Sovereignty）🛰️

“文明层”里有“三步距离”、“动作轻”、“无噪音”。

魔丸的行为： 明明员工离他 5 米远，他为了爽或为了免单，反手就是一个差评说“被跟综了”。

UF 的算法逻辑： 店内的集成传感器（UWB 定位或视觉 AI 节点）会自动记录员工的动线。

判定： 系统调取该时段数据，发现员工全过程保持在 3.5 米 之外。

结果： 系统自动判定该反馈为 “逻辑噪点（Noise）”。这个差评根本不会进入员工的绩效计算池，反而会降低该用户的“文明信用分”。

2. 众议与统计学保护（Bayesian Filtering）🛡️

逻辑： 1/5 效能意味着我们不看“单次偶发”，我们看“结构稳定”。

算法设计： 如果一个空间今天有 100 个用户，99 个反馈都是“极度放松（Relaxed）”，只有这一个“魔丸”说“被打扰了”。

判定： 算法会自动识别出这是一个 “离群值（Outlier）”。

保护： 这种单次的恶念，连员工的“文明奖金”小数点都动不了。系统会自发过滤掉这个垃圾信息。

3. “魔丸隔离”协议：我们要有“拒客”的勇气 🚫

这是UF跟别的品牌最大的不同。

传统逻辑： 顾客是上帝，魔丸也是上帝。员工受委屈，还得赔礼道歉。

UF 逻辑： 空间是为了“文明人”设计的。

如果一个用户在系统里连续两次触发“恶意虚假反馈”或“高维噪音行为”。

App 操作： 系统自动将该用户的权限降级，甚至在下次他想预约进店时提示：“对不起，您的审美逻辑与本空间暂不匹配，请移步。”

对员工的治愈： 员工看到系统自动帮他挡住了“魔丸”，这种 “逻辑层面的撑腰”，就是“治愈型反馈”。
